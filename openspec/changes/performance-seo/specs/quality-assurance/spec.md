## MODIFIED Requirements

### Requirement: Static Code Quality Analysis
The system SHALL employ Biome to format and lint all source files, enforcing alphabetical import sorting and zero-unused-import policies. The `npx biome check ./src` command SHALL exit with zero diagnostics on every CI run.

#### Scenario: Alphabetical Import Check
- **WHEN** the `biome check` command is executed
- **THEN** it SHALL flag any imports or exports that are not alphabetically ordered within their respective groups.

#### Scenario: Zero Unused Imports
- **WHEN** an import statement is not referenced in the component logic or template
- **THEN** Biome SHALL report an error and, if run with `--write`, automatically remove the statement.

#### Scenario: Clean CI Gate
- **WHEN** a pull request CI pipeline executes `npx biome check ./src`
- **THEN** the command SHALL exit with code 0 and produce zero diagnostic messages.

## ADDED Requirements

### Requirement: Test Coverage Baseline
The project SHALL maintain a documented test coverage baseline generated by Vitest with the `@vitest/coverage-v8` provider.

#### Scenario: Coverage report generation
- **WHEN** `npx vitest run --coverage` is executed
- **THEN** it SHALL produce an LCOV report in `coverage/lcov.info` and a text summary to stdout.

#### Scenario: Coverage baseline documentation
- **WHEN** the coverage report is generated for the first time
- **THEN** the line/branch/function coverage percentages SHALL be documented in the project README or a dedicated `COVERAGE.md`.

### Requirement: Lighthouse Production Audit
A Lighthouse audit SHALL be run against the production build (`npm run build && npm run preview`) to establish a performance baseline.

#### Scenario: Performance score
- **WHEN** a Lighthouse audit is run against the preview server
- **THEN** the Performance score SHALL be ≥ 95.

#### Scenario: Accessibility score
- **WHEN** a Lighthouse audit is run against the preview server
- **THEN** the Accessibility score SHALL be ≥ 90.

#### Scenario: SEO score
- **WHEN** a Lighthouse audit is run against the preview server
- **THEN** the SEO score SHALL be 100 (given the meta tags from the seo-metadata capability).
