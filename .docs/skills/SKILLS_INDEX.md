# 📚 Project Skill Index & OpenSpec Mapping

This index guides the AI agent on which skills to activate during specific OpenSpec phases for the Premium Portfolio project.

## 🧭 Phase 1: Explore & Propose (Architecture & UI/UX)
*When brainstorming, planning, or creating `proposal.md` and `design.md`.*
- `creative-design/ui-ux-pro-max`: Use for designing the "Finished Product" vs "Blueprint" dual-state theme.
- `creative-design/web-design-guidelines`: Ensure bento-grid layout principles are respected.
- `software-architecture` (Project Bridge): Enforce strict Vue 3 / CSS Variable separation.

## 🛠️ Phase 2: Apply & Tasks (Implementation)
*When writing actual Vue components, TypeScript interfaces, and Tailwind classes.*
- `development/senior-frontend` + `.docs/skills/project-bridge.md`: Core development using Vue 3 Composition API (`<script setup>`).
- `development/typescript-expert`: Enforce strict typing, absolutely no `any`.
- `development/clean-code` & `development/code-reviewer`: Ensure centralized event management and DRY templates.
- `creative-design/tailwind-patterns`: Implement the CSS Grid via Tailwind utilities combined with specific `BentoItem.vue` wrapper components.

## 🧪 Phase 3: Verify & Archive (Testing & Quality)
*When finalizing tasks and ensuring the Definition of Done.*
- `development/test-driven-development`: Write Vitest unit tests for components.
- `development/webapp-testing`: Implement Playwright E2E flows (POM).
- `web-development/web-performance-optimization`: Verify smooth CSS variable transition timing.
- `development/git-commit-helper`: Format final commits.
